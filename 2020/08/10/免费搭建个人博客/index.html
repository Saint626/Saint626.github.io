为什么要写博客

都0202年了，为什么还要写博客？？？

学的东西越多，就越容易忘或者容易搞混，在写博客的过程中也是对知识的进行二次学习，能进行学习上的归纳或者项目上问题的总结，再通过博客的方式分享出来。这样既能帮助到别人也对学习上有一定的帮助就是我坚持写博客的原因。

有CSDN、简书、掘金这些博客平台，为什么还要做个人博客？？？做个人博客的优点是：网站可以随意定制，可以任意折腾自己的网站，想怎么搞就怎么搞，不用担心由于平台规则啥的被管理员删除。

搭建Hexo个人博客

本文只介绍如何使用hexo + GitHub搭建一个完全免费的个人博客，只是搭建的过程稍微有点复杂，但是搭建好了以后基本不用为如何维护而操心。

环境搭建

因为Hexo是基于Node.js的，安装依赖需要使用npm，而Node.js中自带了npm，并且还需要使用git来托管hexo到github pages服务来托管个人博客。

  1. Node.js：https://nodejs.org/zh-cn/
  2. Git：https://git-scm.com/downloads

下载安装基本都是直接下一步，按照默认的配置完成安装

安装完成后，win+x+a打开powershell，分别输入node -v、npm -v、git --version来查看环境变量是否自动配置好了

安装Hexo

在桌面或者其他位置创建一个文件夹，如 G:\myblog

进入该文件夹，然后右键打开git bash，使用npm或cnpm命令安装hexo

安装完成后，需要初始化博客

hexo init是在当前文件夹下初始化hexo，hexo init blog是在myblog文件夹下新建一个blog文件夹作为hexo初始化文件夹

然后分别输入以下命令，在本地启动hexo

看命令行中提示，hexo 已经运行在 http://localhost:4000，按 ctrl + C 停止hexo

在浏览器中输入 http://localhost:4000，即可查看hexo个人博客了

Github

https://github.com/

使用邮箱注册创建Github的账户

在桌面右键，选择 Git Bash Here， 设置传到github上的用户名和邮箱

创建SSH密钥（可省略）

一直回车，会返回一个路径提示，这是保存ssh密钥的路径，如：

用记事本或者notepad或vscode打开id_rsa.pub文件，复制里面所有东西。

登录到Github，点击右上角头像，进入setting

选择左侧的SSH and GPG keys，点击New SSH key，Title是密钥名，随便取一个名字就红了，key就是刚刚复制的东西，粘贴到里面就好了，点击Add SSH key添加完成

查看是否连接上

桌面右键打开git bash，输入ssh -T git@github.com 出现Are you sure……输入yes回车确认，如果有successfully就是链接成功了

创建Github Pages仓库

右上角点击加号然后new repository

repository name输入用户名.github.io，这个一定要写用户名，github会自动为你打开这个github pages的功能

直接选择create repository，博客的地址就是：https://用户名.github.io

配置hexo部署到GitHub

进入刚刚创建的仓库

打开本地hexo初始化目录下的_config.yml，如：G:\Desktop\myblog\myblog\_config.yml，修改其中的deploy中的值，repo就是刚刚复制的链接放在这里就好了

安装git部署插件

还是在hexo根目录下G:\Desktop\myblog\myblog打开git bash，然后输入三条命令

GitHub有时候在国内会有延迟，过个1、2分钟就可以在浏览器输入刚刚创建的仓库名，我的是saint626.github.io，直接在浏览器输入这个就好了，然后就可以看到你的本地hexo已经在GitHub pages托管了，可以在线访问了。
